# Local Skills

This directory holds reusable, project-scoped Skills that pair well with the
[`PROMPT.md`](../../PROMPT.md) monorepo blueprint. Each subdirectory contains a
`SKILL.md` describing the skill's purpose, triggers, and usage; some include
helper `scripts/`, `references/`, or `data/`.

## Bundled Skills

| Skill | Purpose |
|---|---|
| `api-design-principles` | REST/GraphQL API design guidance and review checklist. |
| `architect` | System design and refactor planning for the TS monorepo (React + NestJS + Prisma). |
| `build-fixer` | Minimal-diff TypeScript / build error resolution. |
| `code-reviewer` | Bug, regression, boundary, a11y, theme, and security review for changed code. |
| `context7` | Fetch up-to-date library docs via the Context7 API. |
| `docs-manager` | Maintain `docs/` and `CLAUDE.md`; sync docs with code. |
| `frontend-design` | Distinctive, production-grade UI/CSS guidance (artifacts and in-repo). |
| `i18n-guide` | Add/audit translations and keep `en.ts` / `zh.ts` in sync. |
| `prisma-migration-guardian` | Keep the Prisma migration chain clean across branches. |
| `security-reviewer` | OWASP-style vulnerability scan and remediation for NestJS + Prisma + React. |
| `skill-creator` | Author or update Skills using the standard layout. |
| `sync` | Safely sync local changes with remote. |
| `ui-ux-pro-max` | Searchable UI/UX design intelligence with a local `local` stack. |

## Adapting To Your Project

These skills use `@<scope>/*` as a placeholder for the workspace's package
scope (`@<scope>/web`, `@<scope>/server`, `@<scope>/shared`, `@<scope>/sdk`,
`@<scope>/ui`). When you adopt the skills in a real project, do a global
search-and-replace of `@<scope>` with your actual scope (e.g. `@my-app`).

For `ui-ux-pro-max`, the repo-baseline dataset lives at
[`ui-ux-pro-max/data/stacks/local.csv`](ui-ux-pro-max/data/stacks/local.csv)
and the matching reference at
[`ui-ux-pro-max/references/local-ui.md`](ui-ux-pro-max/references/local-ui.md).
Edit both to reflect your project's stack, tokens, and primitives before
invoking the skill with `--stack local`.

## Conventions

- Each skill is self-contained under its own folder.
- The `SKILL.md` frontmatter (`name`, `description`) is what the harness uses
  to decide when to activate a skill -- keep descriptions specific.
- Scratch files generated by skills are git-ignored via the local
  [`.gitignore`](.gitignore).
