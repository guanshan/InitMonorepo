# Accessibility Guidelines

- Keep labels attached to form controls; do not rely on placeholders as the only cue.
- Use Radix primitives for dialog focus trapping and keyboard escape behavior.
- Preserve visible focus treatment through semantic token-based outlines and glows.
- Keep status feedback accessible: loading states, empty states, and request toasts should remain readable without color alone.
- Storybook a11y checks should stay enabled for shared UI primitives.
