# Local Skills

This directory holds the project-scoped Skills bundled with `real-demo`. Each
subdirectory contains a `SKILL.md` describing the skill's purpose, triggers, and
usage; some include helper `scripts/`, `references/`, or `data/`.

These skills have been **adapted to this demo's stack** (React 19 + React Router
v7 + NestJS 11 + Prisma + MySQL/Redis, package scope `@real-demo`). They are
loaded by Claude Code via the `.agents/skills/` convention and surface alongside
the harness defaults.

## Bundled Skills

| Skill | Best for |
|---|---|
| `architect` | Planning new features or refactors — DDD layering on the server, FSD layering on the web. |
| `api-design-principles` | Designing or reviewing endpoints; enforces the `{ success, data, meta }` envelope, stable `operationId`s, and the `packages/shared` → server → SDK pipeline. |
| `build-fixer` | Making `make build` / `make typecheck` green again with minimal diffs. |
| `code-reviewer` | End-of-change review: bugs, regressions, DDD/FSD boundaries, theme tokens, a11y, testing gaps. References `docs/*-guidelines.md`. |
| `context7` | Fetching up-to-date library docs through the Context7 API (lower priority than the global `ctx7` CLI workflow if available). |
| `docs-manager` | Maintaining `docs/` and the `CLAUDE.md` index after code or contract changes. |
| `frontend-design` | Distinctive, production-grade UI work. Already aware of `packages/ui/src/tokens.css`, CSS Modules, and the no-Tailwind rule. |
| `i18n-guide` | Adding/auditing translations across `apps/web/src/locales/en.ts` and `zh.ts`. |
| `prisma-migration-guardian` | Keeping the `apps/server/prisma` migration chain clean across branches. |
| `security-reviewer` | OWASP-style scan focused on the NestJS + Prisma + React surface. |
| `skill-creator` | Authoring or updating skills using the standard layout. |
| `sync` | Safely syncing local changes with the upstream of the current branch. |
| `ui-ux-pro-max` | Searchable UI/UX intelligence. The `local` stack in `ui-ux-pro-max/references/local-ui.md` is already pinned to this repo (React 19 + RR v7 + CSS Modules + semantic tokens). |

## Picking The Right Skill

There is overlap between several review-style skills. Rough decision tree:

- Designing a new endpoint → `api-design-principles`
- Designing a new module or rethinking boundaries → `architect`
- After writing code, before merge → `code-reviewer`
- Specific security concerns (auth, secrets, user input) → `security-reviewer`
- UI quality / visual direction → `frontend-design` + `ui-ux-pro-max`
- Build is red → `build-fixer`
- Translations missing / out of sync → `i18n-guide`
- Prisma migrations look weird after a sync → `prisma-migration-guardian`

## Reusing These Skills Outside `real-demo`

If you copy this directory into another project, the package scope (`@real-demo`)
and module names (`users`, `models`, `providers`, etc.) will need to be replaced
to match the new project's reality. The data files under
[`ui-ux-pro-max/data/`](ui-ux-pro-max/data/) and the local stack reference at
[`ui-ux-pro-max/references/local-ui.md`](ui-ux-pro-max/references/local-ui.md)
should also be updated to reflect the target project's tokens and primitives
before invoking `ui-ux-pro-max --stack local`.

## Conventions

- Each skill is self-contained under its own folder.
- The `SKILL.md` frontmatter (`name`, `description`) is what the harness uses to
  decide when to activate a skill — keep descriptions specific.
- Scratch files generated by skills are git-ignored via the local
  [`.gitignore`](.gitignore).
- When introducing new skills, prefer pointing them at this repo's real files
  (`docs/*-guidelines.md`, `packages/shared/src/contracts/api-envelope.ts`,
  `packages/ui/src/tokens.css`, etc.) instead of generic templates.
